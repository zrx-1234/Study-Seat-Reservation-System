## 学生端用户故事 ##

1. 座位查询与搜索

用户故事 1：查看可用座位
As a 需要自习的学生 
I want to 查看当前所有可用的自习室与座位信息 
So that 我可以快速找到可以使用的学习空间
情境说明
	•	系统需实时显示座位状态
	•	显示教室名称、座位编号、可用时间
验收条件
	•	能查看所有开放中的自习室
	•	每个座位状态（可用/已预约）清晰标示

用户故事 2：条件筛选座位
As a 有特定需求的学生 
I want to 根据条件（时间、位置、插座）筛选座位 
So that 我可以找到最符合需求的座位
情境说明
	•	支持「靠窗」「靠插座」等筛选
	•	支持时间区间筛选
验收条件
	•	可进行多条件组合筛选
	•	带插座座位需有明显标识

2. 座位预约

用户故事 3：预约座位
As a 想安排学习时间的学生 
I want to 选择座位和时间进行预约 
So that 我可以确保到场时有座位可用
情境说明
	•	以整点时间为单位预约
	•	单次最多预约4小时
验收条件
	•	不可预约已被占用的时间段
	•	预约成功后生成记录

用户故事 4：预约时间限制
As a 学生 
I want to 每次预约有时间上限 
So that 所有学生都能公平使用资源
验收条件
	•	单次预约不超过4小时
	•	系统支持调整上限参数

3. 签到机制

用户故事 5：完成签到
As a 已预约座位的学生 
I want to 到达后完成签到 
So that 系统确认我实际使用该座位
情境说明
	•	Web端：输入动态编码
	•	微信端：扫描二维码
验收条件
	•	动态码每日更新
	•	编码仅对对应教室有效

4. 提醒与违约管理

用户故事 6：预约提醒
As a 学生 
I want to 在预约开始前收到提醒 
So that 我不会忘记预约时间
验收条件
	•	预约前15分钟发送提醒
	•	支持邮件或微信通知

用户故事 7：未签到处理机制
As a 系统 
I want to 对未签到的预约自动处理 
So that 提高座位利用率
流程说明
	•	+10分钟：发送提醒
	•	+15分钟：自动取消预约
验收条件
	•	自动释放座位
	•	记录违约次数
	•	通知用户预约取消

5. 历史预约管理

用户故事 8：查看历史记录
As a 学生 
I want to 查看我的历史预约记录 
So that 我可以管理自己的使用情况
验收条件
	•	显示状态（已完成/已取消/违约）
	•	支持按时间排序

用户故事 9：再次预约
As a 有固定学习习惯的学生 
I want to 快速再次预约历史座位 
So that 提高使用效率
