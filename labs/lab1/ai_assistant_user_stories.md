# 学习座位预约系统 - AI 智能化功能用户故事 (AI Assistant User Stories)

## 模块一：自然语言交互查询

### US-AI-01：实时空座语义查询
- **用户故事**：作为一个想偷懒的学生，我想要直接用自然语言（如“今晚还有座吗？”）询问助手，以便于我不需要在列表里翻来翻去。
- **验收条件**：
    - 系统能识别时间词（今天、晚上、下午等）。
    - 助手能解析意图并返回具体教室、座位号。
    - 若无空位，助手能给出友好的提示。

### US-AI-02：复杂条件特征搜索
- **用户故事**：作为一个对座位有要求的学生，我想要告诉助手“找个靠窗且带插座的座位”，以便于满足我长时间学习的需求。
- **验收条件**：
    - 助手能提取“靠窗”、“插座”等多个特征属性。
    - 搜索结果必须同时满足用户提到的所有物理标记。

## 模块二：个性化行程与管理

### US-AI-03：个人预约快速查询
- **用户故事**：作为一个健忘的学生，我想要询问“我今天定了哪里的座位”，以便于快速找到我的自习地点。
- **验收条件**：
    - 系统需自动关联当前登录的用户 ID。
    - 准确返回该用户当天的预约状态（时间、地点、座位号）。

### US-AI-04：多轮对话与纠错
- **用户故事**：作为一个用户，我希望助手能记得我上一句话说了什么，以便于进行连续提问（如先问“哪里有座”，再问“那靠窗的呢”）。
- **验收条件**：
    - 助手具备基础的对话历史维护能力。
    - 能够处理模糊表达，并在必要时反问用户以确认需求。
