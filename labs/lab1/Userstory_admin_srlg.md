# 学习座位预约系统 - 管理端用户故事 (User Stories)

## 模块一：基于角色的访问控制 (RBAC) 权限管理



**US-A01: 角色维护**
- **用户故事**: 作为一个超级管理员，我想要能够在系统中创建、编辑和删除不同的管理角色，以便于我能根据学校的管理架构（如：校级管理员、院系管理员）划分权限层次。
- **验收标准**:
  - 系统提供角色管理页面。
  - 支持新增角色并为其命名。
  - 支持修改或删除现有角色（如角色下已有用户，需提示转移或阻止删除）。

**US-A02: 角色分配**
- **用户故事**: 作为一个超级管理员，我想要能够给不同的后台用户（老师或工作人员）分配特定的角色，以便于他们获得与其职责相匹配的操作权限。
- **验收标准**:
  - 在用户管理界面，可以通过下拉菜单或勾选框为用户绑定角色。
  - 一个用户可以支持绑定多个角色。

**US-A03: 动态菜单展示**
- **用户故事**: 作为一个管理端用户，我想要系统界面仅展示我被授权操作的功能菜单，以便于我不会被无关功能干扰，同时保证系统的安全性。
- **验收标准**:
  - 用户登录后，前端根据其角色权限动态渲染侧边栏菜单。
  - 尝试通过直接输入URL访问未授权页面时，系统需拦截并提示无权限。

## 模块二：自习室与座位资源管理

**US-A04: 自习室登记与注销**
- **用户故事**: 作为一个管理员，我想要能够登记新的自习室或注销废弃的自习室，并能设置其归属（全校通用或指定院系专用），以便于我能灵活控制全校的可用自习空间。
- **验收标准**:
  - 支持录入自习室名称、位置、容量等基本信息。
  - 支持设置自习室属性：公共开放或某院系专属。
  - 被注销的自习室将在学生端不可见，且自动取消该室内未来的所有预约。

**US-A05: 自习室开放时间设置**
- **用户故事**: 作为一个管理员，我想要能够设定每个自习室的每日开放时间（如默认的早7点到晚10点，或个别通宵教室），以便于系统能精确控制学生的可预约时间段。
- **验收标准**:
  - 支持按单个自习室或批量设置开放和关闭时间。

**US-A06: 座位登记、注销与可用性维护**
- **用户故事**: 作为一个管理员，我想要能够对自习室内的具体座位进行登记、生成唯一编号，或在座位损坏时注销/暂时禁用该座位，以便于向学生提供准确的可用座位清单。
- **验收标准**:
  - 支持单个或批量生成座位号。
  - 支持更改座位状态（正常可用、维修中、已注销）。

**US-A07: 座位特殊标记维护**
- **用户故事**: 作为一个管理员，我想要能够为特定座位添加属性标记（如“靠近固定插座”、“移动导轨插座”），以便于学生在找座位时可以通过这些条件进行精确搜索。
- **验收标准**:
  - 在座位编辑页面，可以勾选相应的硬件设施标签。

## 模块三：预约与违约监督

**US-A08: 预约记录全局查看**
- **用户故事**: 作为一个管理员，我想要能够按时间、自习室、学生学号等条件检索并查看所有预约记录，以便于我掌握各个自习室的真实使用率和周转率。
- **验收标准**:
  - 提供数据表格展示预约记录（预约人、座位号、时间段、签到状态）。
  - 支持多条件组合查询和分页显示。

**US-A09: 违约记录查看与管理**
- **用户故事**: 作为一个管理员，我想要能够查看学生的违约（超时未签到）记录，以便于我能识别恶意占座行为并辅助制定未来的管理惩罚策略。
- **验收标准**:
  - 系统自动记录超15分钟未签到被系统取消的订单，并标记为“违约”。
  - 管理员可导出违约数据列表。

**US-A10: 代理预约与取消**
- **用户故事**: 作为一个管理员，我想要能够直接在后台为指定用户进行座位的预约或取消预约，以便于在系统故障、特殊活动征用或学生遇到突发困难时进行人工干预。
- **验收标准**:
  - 后台支持输入学生学号直接锁定某个座位的时间段。
  - 管理员取消学生预约时，可选择是否发送系统通知告知学生原因。

## 模块四：系统全局参数调整

**US-A11: 预约规则参数动态调整**
- **用户故事**: 作为一个超级管理员，我想要能够在后台灵活调整诸如“单次最大预约小时数（如默认4小时）”等系统参数，以便于我能根据不同时期（如期末考试周）的座位紧张程度，动态优化资源分配。
- **验收标准**:
  - 提供全局参数设置页面。
  - 修改参数保存后即时生效，对修改后产生的新预约订单按新规则执行。
